# Phase 8: Usuários, Permissões, Notificações e Relatórios

<objective>
Implementar os endpoints de backend (schemas, controllers e routes) para gestão de Usuários, Tags, Categorias, Notificações e Relatórios, integrando o envio automático de notificações ao fluxo de aprovação de empréstimos.
</objective>

<threat_model>
- **T-08-01:** Escalada de privilégios via edição de profile (BUSER-03).
  - *Mitigation:* Validar se o usuário que edita é admin ou o próprio dono. Se for o dono, ignorar tentativas de alterar `role` ou `tagId`.
- **T-08-02:** Manipulação de notificações alheias (BNOTIF-02).
  - *Mitigation:* Em `PUT /api/notifications/:id`, forçar a cláusula `where: { userId: req.user.id }` via ORM.
- **T-08-03:** Acesso indevido a KPIs de relatórios.
  - *Mitigation:* Proteger `/api/reports/*` com `requirePermission('gerar_relatorios')` e `requireAuth`.
</threat_model>
